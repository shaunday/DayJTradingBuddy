Reference to the db context project, add ef core, ef core tools, ef sql

1. 
_Create Migration class - has all the info about how to create the DB based on the model
Package Manager Console: EntityFrameworkCore\Add-Migration InitialCreate -p JTA.Infrastructure -s JTA.Infrastructure
* The -s flag is optional in this case because your design-time factory already handles the configuration.
**without "EntityFrameworkCore\" you'll get errors

_Apply Migration
2a. Directly
EntityFrameworkCore\Update-Database -p JTA.Journal.Repository -s JTA.Web.API

2b. Script
save the script in /db-scripts : DATE_WHAT_CHANGED
Document what changed... release notes of sorts = Version
Then apply the script in the DB/ do 2a.

PM: EntityFrameworkCore\Script-Migration -Idempotent -Output "db-scripts\14_12_2024_Initial.sql" -Project JTA.Journal.Repository -StartupProject JTA.Web.API
CLI: dotnet ef migrations script --idempotent -o db-scripts\14_12_2024_Initial.sql -p JTA.Journal.Repository -s JTA.Web.API


(Script needs to be applied to the DB))
